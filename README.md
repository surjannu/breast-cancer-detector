## Project Overview

This project builds an end-to-end machine learning system for **breast cancer diagnosis**. It uses the [UCI Breast Cancer Wisconsin (Diagnostic) dataset](https://archive.ics.uci.edu/dataset/17/breast+cancer+wisconsin+diagnostic) to train multiple classifiers that predict whether a tumour is **Malignant** or **Benign** based on features extracted from digitised fine-needle aspirate (FNA) images of breast masses.

The entire system runs with a **single command**. It downloads the data, cleans it, trains five different models, selects the best performer, generates diagnostic plots, and launches an interactive Streamlit dashboard where clinicians or analysts can:

- Explore the dataset visually
- Compare all model performance side-by-side
- Enter new patient measurements and receive an instant prediction with probability score

---

## Business Problem

Breast cancer is the most common cancer in women worldwide. Early and accurate diagnosis dramatically improves survival rates — the 5-year survival rate for localised breast cancer is over 99%, dropping to ~29% for distant-stage disease.

Current diagnostic workflows often rely on manual visual interpretation of biopsy images, which can be time-consuming and subject to human error. A reliable ML classifier can:

- **Reduce misdiagnosis rates** — false negatives (missed malignant cases) delay treatment with serious consequences
- **Assist radiologists** with a second opinion, especially in resource-constrained settings
- **Prioritise cases** for specialist review

**Cost asymmetry:**

| Outcome | Consequence |
|---|---|
| False Negative (missed malignancy) | Delayed treatment; significantly worse prognosis |
| False Positive (benign called malignant) | Unnecessary follow-up biopsy; patient anxiety |

Given this asymmetry, models are evaluated with emphasis on **high recall** for malignant cases alongside overall F1-Score and ROC-AUC.

## Dataset Details

| Attribute | Value |
|---|---|
| Source | [UCI ML Repository — Breast Cancer Wisconsin (Diagnostic)](https://archive.ics.uci.edu/dataset/17/breast+cancer+wisconsin+diagnostic) |
| Instances | 569 |
| Features | 30 numeric |
| Target classes | Malignant (M) = 212 samples · Benign (B) = 357 samples |
| Missing values | None |
| Year | 1995 — W.N. Street, W.H. Wolberg, O.L. Mangasarian |

**Feature groups** — each of the 10 cell-nucleus measurements is summarised with 3 statistics = 30 total features:

| Group | Features |
|---|---|
| Mean | radius, texture, perimeter, area, smoothness, compactness, concavity, concave points, symmetry, fractal dimension |
| Standard Error | same 10 features |
| Worst (max) | same 10 features |

## Model Development

Five classifiers are trained and compared:

| # | Model | Notes |
|---|---|---|
| 1 | **Logistic Regression** | Baseline linear model, `lbfgs` solver, max_iter=1000 |
| 2 | **Random Forest** | 100 estimators, majority-vote ensemble |
| 3 | **SVM (RBF kernel)** | Margin-based, probability calibration enabled |
| 4 | **XGBoost** | Gradient boosting, 100 rounds |
| 5 | **K-Nearest Neighbours** | k=5, distance-based |

All models are trained on features **standardised with `StandardScaler`** (fit on training data only, applied to both train and test sets). Model selection uses **F1-Score** on the held-out test set (80/20 stratified split, `random_state=42`).

## Pipeline Architecture

```
run_pipeline.py
│
├── Step 1 ─ Data Download       (src/data/download.py)
│            └─ ucimlrepo → sklearn fallback → data/raw/breast_cancer.csv
│
├── Step 2 ─ Preprocessing       (src/data/preprocess.py)
│            └─ clean, encode, scale, split → data/processed/
│
├── Step 3 ─ EDA Visualisations  (src/visualization/plots.py)
│            └─ class dist, correlation heatmap → reports/figures/
│
├── Step 4 ─ Model Training      (src/models/train.py)
│            └─ 5 classifiers → models/{name}.pkl
│
├── Step 5 ─ Evaluation          (src/models/evaluate.py)
│            └─ metrics table, best model → models/best_model.pkl
│                                         → reports/model_comparison.csv
│
└── Step 6 ─ Performance Plots   (src/visualization/plots.py)
             └─ confusion matrix, ROC curves, feature importance → reports/figures/
```

## Local Setup and Running

### Prerequisites

- Python 3.10 or higher
- pip

### 1 — Clone the repository

```bash
git clone https://github.com/surjannu/breast-cancer-detector.git
cd breast-cancer-detector
```

### 2 — Install dependencies

```bash
pip install -r requirements.txt
```

### 3 — Run the full pipeline

```bash
python run_pipeline.py
```

This single command will:
1. Download the UCI Breast Cancer Wisconsin dataset (via `ucimlrepo`; falls back to `sklearn` if unavailable)
2. Clean and preprocess the data
3. Generate EDA visualisations
4. Train all five classifiers
5. Evaluate and select the best model
6. Save all outputs to `models/`, `data/processed/`, and `reports/figures/`

### 4 — Launch the interactive dashboard

```bash
streamlit run app.py
```

Then open `http://localhost:8501` in your browser.

## Conclusion and Future Directions

All five models achieve **>95% accuracy** on the UCI breast cancer dataset, with the top three reaching 97.4% accuracy and ROC-AUC scores above 0.99. The Random Forest and XGBoost models are especially competitive.

**Potential future enhancements:**

- Hyperparameter tuning with Optuna or `GridSearchCV`
- SHAP explainability values integrated into the dashboard for per-prediction feature attribution
- Clinical decision threshold tuning to maximise recall for the malignant class
- Deployment as a REST API (FastAPI + Docker)
- Integration with actual FNA image data using CNNs
- Cross-validation results and learning curves
